# Security Policy

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

To report a security vulnerability, email **harun@s4e.io**.

Include as much detail as possible:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix

We will respond to your report within 48 hours and keep you informed throughout the resolution process. Please do not disclose the vulnerability publicly until it has been fixed and announced.

## Scope

This policy covers the Opservant Spark agent (`github.com/s4e-io/opservant-spark`) including all released versions.

For playbook-related security issues, see [opservant-playbooks](https://github.com/s4e-io/opservant-playbooks/blob/main/SECURITY.md).

## Supported Versions

Security fixes are applied to the latest release only.

## Disclosure Policy

Once a fix is released, we will publish a security advisory on GitHub. We ask that you give us reasonable time to fix and release before any public disclosure.
